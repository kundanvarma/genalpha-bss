The mobile app

Design proposal — “MyGenAlpha / MyNova”: one care-first app, white-labeled per tenant · React Native (Expo)

Design principles

  • Care first, shop second. Customers open a telco app to check usage, pay a bill or get help — those are one tap from launch. Buying more is present but never in the way.
  • One binary, every operator. The app resolves its tenant at first launch (same gateway mechanism as the web channels) and pulls branding tokens — name, logo, colors. The demo: switch operator in settings and the whole app rebrands and re-authenticates against the other realm.
  • Nothing new on the backend. Every screen below is powered by APIs that exist and are E2E-tested today. The app is a fourth channel, not a new system.
  • Same auth model: OIDC authorization-code + PKCE in the system browser, against the tenant's issuer. Biometric unlock and push notifications are later additive steps.

Build approach

Expo (React Native). One codebase targeting iOS, Android and web. The web target is the autonomous-verification trick: the whole app gets a Playwright E2E suite exactly like the five web-channel suites, runnable tonight without Xcode or Android SDKs. When native tooling is available on the Mac, the identical code runs on simulators/devices — nothing is rewritten. Phase 2 (native-only): push notifications from the TMF681 stream, biometric unlock.
